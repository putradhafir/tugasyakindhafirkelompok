LATAR BELAKANG & PENGGUNAAN APLIKASI

Kepadatan kendaraan dan kurangnya sistem parkir terintegrasi menjadi masalah utama di area perkotaan. Smart Parking Management hadir sebagai solusi cerdas menggunakan pendekatan Object Oriented Programming (OOP) untuk memodelkan kendaraan, slot parkir, transaksi, dan petugas secara real-time. Dengan OOP, sistem ini mudah dikembangkan, mendukung berbagai tipe kendaraan, dan memberikan transparansi ketersediaan slot.

Cara penggunaan aplikasi: Sistem ini dirancang untuk petugas parkir dan manajer. Petugas dapat memantau ketersediaan slot, menempatkan kendaraan, dan membebaskan slot. Transaksi otomatis menghitung biaya berdasarkan durasi. Manajer dapat melihat laporan okupansi per zona. Aplikasi ini dapat diintegrasikan dengan pintu otomatis dan kamera untuk identifikasi plat nomor.

Mengapa OOP? & Objek Smart Parking

Pendekatan OOP memetakan entitas dunia nyata (kendaraan, slot, transaksi) menjadi objek yang memiliki data dan perilaku masing-masing. Hal ini mengurangi duplikasi kode, mempermudah perawatan, dan sangat cocok untuk pengembangan fitur seperti reservasi online.

Source 1 - Class Kendaraan (Parent)

class Kendaraan dengan atribut dasar dan method info Ready 
// ============================================================
// Source 1: Class Kendaraan
// Fungsi: Merepresentasikan kendaraan yang parkir.
// Properti:
//   - platNomor : String (nomor polisi)
//   - jenis     : String (jenis kendaraan, misal Mobil/Motor)
//   - waktuMasuk: DateTime (waktu masuk parkir)
// Method:
//   - infoKendaraan() : menampilkan informasi kendaraan
// ============================================================

class Kendaraan {
  // Properti (atribut)
  String platNomor;
  String jenis;
  DateTime waktuMasuk;

  // Constructor
  Kendaraan(this.platNomor, this.jenis, this.waktuMasuk);
  
  // Method untuk menampilkan info
  void infoKendaraan() {
    print("🔹 Kendaraan: $jenis | Plat: $platNomor | Masuk: ${waktuMasuk.hour}:${waktuMasuk.minute.toString().padLeft(2, '0')}");
  }
}

void main() {
  print("=== DEMO CLASS KENDARAAN ===");
  print("=" * 30);
  // Membuat objek kendaraan mobil
  DateTime now = DateTime(2026, 6, 10, 9, 30);
  Kendaraan mobil = Kendaraan("B 2026 XC", "Mobil", now);
  mobil.infoKendaraan();
  
  // Membuat objek kendaraan motor
  Kendaraan motor = Kendaraan("AG 1234 Z", "Motor", DateTime(2026, 6, 10, 10, 0));
  motor.infoKendaraan();
  
  // Membuat objek kendaraan truk
  Kendaraan truk = Kendaraan("N 5678 AB", "Truk", DateTime(2026, 6, 10, 11, 15));
  truk.infoKendaraan();
  
  print("=" * 30);
  print("Total kendaraan terdaftar: 3");
}

Source 2 - SlotParkir (Inheritance)

Pewarisan dari AreaParkir, menunjukkan relasi is-a Ready 
// ============================================================
// Source 2: AreaParkir (base class) dan SlotParkir (turunan)
// AreaParkir: kelas dasar untuk zona parkir.
// SlotParkir: turunan yang menambahkan nomor slot dan status.
// Properti tambahan:
//   - nomorSlot  : int
//   - tersedia   : bool (true jika kosong)
//   - kendaraanPlat : String? (plat kendaraan yang menempati)
// Method:
//   - tempatiSlot(plat) : mengisi slot jika tersedia
//   - bebaskanSlot()    : mengosongkan slot
// ============================================================

class AreaParkir {
  String zona;
  AreaParkir(this.zona);
  void infoZona() => print("📍 Zona parkir: $zona");
}

class SlotParkir extends AreaParkir {
  int nomorSlot;
  bool tersedia;
  String? kendaraanPlat;

  SlotParkir(this.nomorSlot, String zona, this.tersedia) : super(zona);

  // Method untuk menempati slot
  bool tempatiSlot(String plat) {
    if (tersedia) {
      kendaraanPlat = plat;
      tersedia = false;
      print("✅ Slot #$nomorSlot (Zona $zona) ditempati oleh $plat");
      return true;
    } else {
      print("❌ Slot #$nomorSlot sudah terisi!");
      return false;
    }
  }

  // Method untuk membebaskan slot
  void bebaskanSlot() {
    if (!tersedia) {
      print("🆓 Slot #$nomorSlot dibebaskan.");
      kendaraanPlat = null;
      tersedia = true;
    } else {
      print("ℹ️ Slot #$nomorSlot sudah kosong.");
    }
  }
  
  // Method untuk menampilkan status slot
  void infoSlot() {
    String status = tersedia ? "🟢 Tersedia" : "🔴 Terisi (${kendaraanPlat ?? 'Unknown'})";
    print("Slot #$nomorSlot - Zona $zona: $status");
  }
}

void main() {
  print("== DEMO SLOT PARKIR + INHERITANCE ==");
  print("=" * 35);
  
  // Membuat objek SlotParkir dengan zona "A1"
  SlotParkir slot1 = SlotParkir(1, "A1", true);
  slot1.infoZona();  // method dari parent
  slot1.infoSlot();
  
  print("\n--- Proses Parkir ---");
  slot1.tempatiSlot("B 1234 AB");
  slot1.infoSlot();
  
  slot1.bebaskanSlot();
  slot1.infoSlot();
  
  print("\n--- Parkir Kendaraan Lain ---");
  slot1.tempatiSlot("D 9876 ZZ");
  slot1.infoSlot();
  
  print("\n--- Status Akhir ---");
  print("Status tersedia: ${slot1.tersedia}");
  print("Kendaraan di slot: ${slot1.kendaraanPlat ?? 'Kosong'}");
}

Source 3 - Transaksi & Petugas

Billing dan monitoring petugas Ready 
// ============================================================
// Source 3: TransaksiParkir dan PetugasParkir
// TransaksiParkir: menangani pembayaran dan struk.
//   Properti: id, platKendaraan, durasiJam, tarifAwal, tarifPerJam
//   Method: hitungTotal(), cetakStruk()
// PetugasParkir: merepresentasikan petugas dengan nama dan shift.
//   Method: laporan(slotTerisi, totalSlot)
// ============================================================

class TransaksiParkir {
  String id;
  String platKendaraan;
  int durasiJam;
  double tarifAwal = 4000;
  double tarifPerJam = 2000;

  TransaksiParkir(this.id, this.platKendaraan, this.durasiJam);
  
  // Menghitung total biaya berdasarkan durasi
  double hitungTotal() {
    if (durasiJam <= 1) return tarifAwal;
    return tarifAwal + (durasiJam - 1) * tarifPerJam;
  }

  // Mencetak struk parkir
  void cetakStruk() {
    double total = hitungTotal();
    print("🧾 STRUK PARKIR");
    print("═" * 25);
    print("ID Transaksi : $id");
    print("Kendaraan    : $platKendaraan");
    print("Durasi       : $durasiJam jam");
    print("Tarif Awal   : Rp${tarifAwal.toStringAsFixed(0)}");
    print("Tarif/jam    : Rp${tarifPerJam.toStringAsFixed(0)}");
    print("─" * 25);
    print("Total Bayar  : Rp${total.toStringAsFixed(0)}");
    print("═" * 25);
    print("Terima kasih! 🚗");
  }
}

class PetugasParkir {
  String nama;
  String shift;
  
  PetugasParkir(this.nama, this.shift);
  
  void laporan(int slotTerisi, int totalSlot) {
    double okupansi = (slotTerisi / totalSlot) * 100;
    print("📊 LAPORAN PARKIR");
    print("═" * 30);
    print("Petugas      : $nama");
    print("Shift        : $shift");
    print("Slot Terisi  : $slotTerisi / $totalSlot");
    print("Okupansi     : ${okupansi.toStringAsFixed(1)}%");
    if (okupansi > 80) {
      print("⚠️  Status: PARKIR PENUH! Rekomendasi buka area baru.");
    } else if (okupansi > 50) {
      print("🟡 Status: Cukup ramai.");
    } else {
      print("🟢 Status: Tersedia banyak slot.");
    }
    print("═" * 30);
  }
}

void main() {
  print("=== DEMO TRANSAKSI & PETUGAS ===");
  print("=" * 35);
  
  // Membuat transaksi
  TransaksiParkir trans1 = TransaksiParkir("TRX-001", "B 1234 AB", 3);
  trans1.cetakStruk();
  
  print("\n");
  
  TransaksiParkir trans2 = TransaksiParkir("TRX-002", "AG 5678 Z", 1);
  trans2.cetakStruk();
  
  print("\n");
  
  // Membuat petugas dan laporan
  PetugasParkir petugas = PetugasParkir("Budi Santoso", "Pagi (08:00-16:00)");
  petugas.laporan(15, 20);
  
  print("\n");
  
  // Simulasi laporan dengan kondisi berbeda
  PetugasParkir petugas2 = PetugasParkir("Siti Rahayu", "Malam (16:00-00:00)");
  petugas2.laporan(9, 20);
}

class TransaksiParkir {
  String id;

  TransaksiParkir(this.id, this.platKendaraan, this.durasiJam);

class PetugasParkir {
  String nama;
  String shift;
  
  PetugasParkir(this.nama, this.shift);

void main() {
  print("🏢 SMART PARKING SYSTEM - SIMULASI");
  print("=" * 45);
  
  // 1. Inisialisasi Petugas
  print("\n👤 PETUGAS");
  PetugasParkir petugas = PetugasParkir("Ahmad", "Pagi");
  print("Petugas: ${petugas.nama} (Shift ${petugas.shift})");
  
  // 2. Inisialisasi Slot Parkir
  print("\n🅿️ SLOT PARKIR");
  var slots = [
    SlotParkir(1, "A", true),
    SlotParkir(2, "A", true),
    SlotParkir(3, "B", true),
    SlotParkir(4, "B", true),
    SlotParkir(5, "C", true),
  ];
  print("Total slot tersedia: ${slots.length}");
  
  // 3. Inisialisasi Kendaraan
  print("\n🚗 KENDARAAN");
  var kendaraanList = [
    Kendaraan("B 1234 AB", "Mobil", DateTime(2026, 6, 10, 9, 0)),
    Kendaraan("AG 5678 Z", "Motor", DateTime(2026, 6, 10, 10, 15)),
    Kendaraan("D 9876 ZZ", "Mobil", DateTime(2026, 6, 10, 11, 30)),
    Kendaraan("N 1111 AA", "Truk", DateTime(2026, 6, 10, 13, 45)),
  ];
  
  // 4. Proses Parkir
  print("\n🔄 PROSES PARKIR");
  int slotTerisi = 0;
  var transaksiList = [];
  
  for (var i = 0; i < kendaraanList.length && i < slots.length; i++) {
    var kendaraan = kendaraanList[i];
    var slot = slots[i];
    print("\n--- Parkir Kendaraan ${i+1} ---");
    kendaraan.infoKendaraan();
    if (slot.tempatiSlot(kendaraan.platNomor)) {
      slotTerisi++;
      // Buat transaksi
      int durasi = (i + 1) * 2; // Durasi bervariasi
      var transaksi = TransaksiParkir(
        "TRX-${(i+1).toString().padLeft(3, '0')}", 
        kendaraan.platNomor, 
        durasi
      );
      transaksiList.add(transaksi);
    }
  }
  
  // 5. Tampilkan laporan
  print("\n📋 LAPORAN AKHIR");
  petugas.laporan(slotTerisi, slots.length);
  
  // 6. Tampilkan transaksi
  print("\n💰 TRANSAKSI");
  for (var trans in transaksiList) {
    trans.cetakStruk();
    print("");
  }
  
  print("=" * 45);
  print("✅ Simulasi selesai!");
}

KESIMPULAN

Dengan pendekatan Object Oriented Programming (OOP), sistem Smart Parking Management ini berhasil memodelkan entitas parkir secara terstruktur dan modular. Keunggulan yang diperoleh:

  • ✅ Modularitas: Setiap class memiliki tanggung jawab yang jelas (Kendaraan, SlotParkir, Transaksi, Petugas).
  • ✅ Reusability: Class dapat digunakan kembali dan dikembangkan (contoh: inheritance pada SlotParkir).
  • ✅ Maintainability: Perubahan pada satu class tidak mempengaruhi class lain.
  • ✅ Real-world Modeling: Objek mencerminkan entitas nyata di sistem parkir.
  • ✅ Scalability: Mudah menambahkan fitur baru seperti reservasi online atau integrasi IoT.

Implementasi ini menunjukkan bahwa OOP adalah pendekatan yang tepat untuk membangun sistem manajemen parkir yang cerdas, efisien, dan mudah dikembangkan di masa depan.

Insight: Sistem ini dapat diintegrasikan dengan teknologi IoT seperti sensor ultrasonik untuk deteksi slot secara real-time, kamera untuk identifikasi plat nomor otomatis, dan aplikasi mobile untuk reservasi.
